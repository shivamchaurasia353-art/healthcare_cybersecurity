Vendor Authentication

Seed Hint

Run the seed script first:
node backend/scripts/seed.js

It will print API keys for Apollo Hospital and CityLab Diagnostics and the demo patient's Health ID.

Then paste the API key above and the Health ID in the consent request.

Step 2 — Request Patient Consent
As a vendor, request access to a patient's health records. The patient will receive a notification to approve or deny.
Prescription Lab Report Discharge Summary Imaging Vaccination Insurance Consultation
Step 3 — Patient Approval
Approve or deny the consent request on behalf of the patient for demo purposes, or switch to the patient portal to do it manually.
⚡ Demo Quick Action — Approve or Deny as Patient
Manual check / patient portal

Or open localhost:3000 → Consent tab → Approve manually

Step 4 — Consent Token
Once approved, a cryptographically signed consent token is issued. This token proves the patient's authorization.
Step 5 — Access Patient Records
Use the consent token to securely fetch the patient's health records. Access is scope-limited to approved data types only.
